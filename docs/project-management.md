# Gestión del proyecto

## Tablero Trello

El proyecto se gestiona mediante un tablero Kanban en Trello con las siguientes columnas:

- **Backlog:** todas las tareas pendientes de planificar
- **Todo:** tareas listas para empezar
- **In Progress:** tareas en desarrollo activo
- **Review:** tareas terminadas pendientes de revisión
- **Done:** tareas completadas

## ¿Por qué Kanban y no Scrum?

He elegido Kanban porque el proyecto es individual y el trabajo fluye de forma continua sin necesidad de sprints fijos. Kanban permite mover tareas según disponibilidad y prioridad, lo que encaja mejor con el ritmo de desarrollo de este proyecto.

## Organización del trabajo

Las tareas se han dividido en estas categorías:

- **Documentación:** agile.md, idea.md, diseño, componentes, hooks, etc.
- **Configuración:** Vite, Tailwind, React Router, estructura de carpetas
- **Frontend:** componentes, páginas, contexto, hooks
- **Backend:** servidor Express, rutas, controladores, servicios
- **Despliegue:** Vercel, variables de entorno
